Ambiente Kubernetes + TLS Automático com CertManager & Let's Encrypt

🎯 Objetivo

Configurar emissão automática de certificados HTTPS usando o CertManager com Let's Encrypt.

🛠 Tecnologias e Ferramentas

  • Kubernetes (OKE Free na Oracle)
  • Helm (Ingress e CertManager)
  • NGINX Ingress Controller
  • CertManager + Let's Encrypt (ACME)
  • Hostinger (domínio)
  • Cloudflare (DNS)
  • Docker & GitHub

📋 Passo a Passo

  1. Instalar o Helm: 
    curl https://raw.githubusercontent.com/helm/helm/main/scripts/get-helm-3 | bash
helm version
  2. Configurar o domínio sredevops.com.br
  3. Instalar o NGINX Ingress Controller: 
    helm repo add ingress-nginx https://kubernetes.github.io/ingress-nginx
helm repo update

helm install ingress-nginx ingress-nginx/ingress-nginx \
  --namespace ingress-nginx \
  --create-namespace
  4. Instalar o CertManager via Helm: 
    helm repo add jetstack https://charts.jetstack.io
helm repo update

kubectl create namespace cert-manager

helm install cert-manager jetstack/cert-manager \
  --namespace cert-manager \
  --version v1.14.4 \
  --set installCRDs=true
  5. Criar um ClusterIssuer com autenticação Cloudflare.
  6. Deploy de uma página HTML/CSS com Docker + HTTPS.

✨ Resultados Alcançados

  • ✅ Imagem Docker publicada no Docker Hub
  • ✅ Código versionado no GitHub
  • ✅ Certificados HTTPS válidos com TLS automático
  • ✅ Página hospedada com domínio próprio + HTTPS
